# 如何设计内核：权限、调度与结构

> 从零理解操作系统内核设计的三个基本模型

操作系统内核的设计围绕三个基本问题展开：**权限**（哪些操作是允许的）、**调度**（谁在什么时候用多久）、**结构**（内核如何组织自己）。本书围绕这三个模型展开，依次讨论每个模型必须回答的根本问题、可能的解决方案以及不同方案之间的权衡。

阅读本书不需要预先具备操作系统专业知识，只需理解程序的基本执行方式、内存的基本组织方式以及中断的基本概念。

**全书共 85 章，分 8 个部分，涵盖从资源管理到设计自己的内核的完整路径。**

| 第 8 部分 | 经典内核设计选择 | 6 章 |

### 快速导航

- **资源模型**（第 2 部分）— 有限、可分配、可竞争：内核管理什么的本质
- **权限模型**（第 3 部分）— 主体→操作→客体：权限的来源、传递、撤销、衰减
- **调度模型**（第 4 部分）— 谁、什么时候、用多久：执行流、中断、同步、电源、迁移
- **结构模型**（第 5 部分）— 分层、模块化、消息传递、可观测性、可扩展性
- **动手设计**（第 6 部分）— 从零开始定义自己的内核
- **架构实战**（第 7 部分）— x86/ARM/RISC-V 的特权级、页表、中断、启动、调试
- **经典对比**（第 8 部分）— Linux、seL4、QNX、L4、HIC 的设计路线
